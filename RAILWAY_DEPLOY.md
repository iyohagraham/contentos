# Deploying Postiz to Railway

This guide will help you deploy Postiz (social media scheduler) to Railway so you can connect real social media accounts.

## Prerequisites

- Railway account (https://railway.app)
- GitHub account
- Social media accounts you want to connect (TikTok, Instagram, YouTube, Facebook)

## Step 1: Deploy Postiz to Railway

1. Go to https://railway.app and sign in
2. Click "New Project" → "Deploy from GitHub repo"
3. Fork the Postiz repository: https://github.com/gitroomhq/postiz-app
4. Select your forked repo
5. Railway will auto-detect it's a Node.js app

## Step 2: Configure Environment Variables

In your Railway project settings, add these environment variables:

### Required Variables

```bash
# Database (Railway will auto-provision PostgreSQL)
DATABASE_URL=postgres://...  # Auto-generated by Railway

# Redis (Railway will auto-provision Redis)
REDIS_URL=redis://...  # Auto-generated by Railway

# Authentication
JWT_SECRET=your-random-secret-key-here
NEXTAUTH_SECRET=your-random-secret-key-here
NEXTAUTH_URL=https://your-app.up.railway.app

# Social Media OAuth (get these from each platform's developer portal)
# TikTok
TIKTOK_CLIENT_KEY=your-tiktok-key
TIKTOK_CLIENT_SECRET=your-tiktok-secret

# Instagram (via Facebook Developer Portal)
INSTAGRAM_CLIENT_ID=your-instagram-id
INSTAGRAM_CLIENT_SECRET=your-instagram-secret

# YouTube (via Google Cloud Console)
YOUTUBE_CLIENT_ID=your-youtube-id
YOUTUBE_CLIENT_SECRET=your-youtube-secret

# Facebook (via Facebook Developer Portal)
FACEBOOK_CLIENT_ID=your-facebook-id
FACEBOOK_CLIENT_SECRET=your-facebook-secret
```

### Generate Secure Secrets

```bash
# Generate random secrets
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

## Step 3: Deploy

1. Railway will automatically deploy when you push to GitHub
2. Wait for deployment to complete (~2-3 minutes)
3. Your app will be available at `https://your-app.up.railway.app`

## Step 4: Connect Social Accounts

1. Open your Postiz app URL
2. Sign up / log in
3. Go to "Channels" or "Integrations"
4. Click "Add Channel" for each platform
5. Follow OAuth flow for each platform
6. Authorize Postiz to post on your behalf

## Step 5: Get API Credentials

1. In Postiz, go to Settings → API
2. Generate an API key
3. Copy the API key and your Postiz URL

## Step 6: Configure ContentOS

Add these to your ContentOS environment:

```bash
# In .env.local (local development)
VITE_POSTIZ_URL=https://your-app.up.railway.app
VITE_POSTIZ_API_KEY=your-postiz-api-key

# In Vercel dashboard (production)
# Add the same variables to your Vercel project
```

## Step 7: Test Connection

1. Restart your ContentOS dev server
2. Go to Channels view
3. You should see "Connected to Postiz" status
4. Your social channels should appear in the list
5. Try creating a video and publishing it

## Troubleshooting

### OAuth Redirect Issues
- Make sure your Railway URL is added to each platform's OAuth redirect URIs
- TikTok: https://developers.tiktok.com
- Instagram: https://developers.facebook.com
- YouTube: https://console.cloud.google.com
- Facebook: https://developers.facebook.com

### Database Connection Issues
- Railway auto-provisions PostgreSQL
- Check that DATABASE_URL is set correctly
- Run migrations if needed: `npm run db:migrate`

### Redis Connection Issues
- Railway auto-provisions Redis
- Check that REDIS_URL is set correctly
- Test connection: `redis-cli -u $REDIS_URL ping`
